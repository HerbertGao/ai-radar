## MODIFIED Requirements

### 需求:飞书自定义机器人通道推送

系统必须提供一个飞书推送通道，基于飞书自定义机器人 webhook（原生 fetch + 签名）把每日日报/产品发现/告警推送到配置的飞书群。**飞书是可选通道，不破坏 P1 纯 Telegram 部署的向后兼容**：未配置飞书（无 `FEISHU_WEBHOOK_URL` / `FEISHU_SIGN_SECRET`）时，飞书**不纳入「已配置通道」集**、不参与分发、不报错——「向所有已配置通道分发」（daily-intel-pipeline）据此只发已配置的通道（至少 Telegram）。仅当**配置不完整**（声明启用飞书但 webhook URL 或签名密钥缺其一）时，env 校验必须以明确错误快速失败，禁止用空值发送或静默半启用。消息必须渲染为飞书 JSON 卡片（区别于 Telegram 的 MarkdownV2），卡片内跳转只用按钮/文字链方式（飞书自定义机器人不支持点击回调到服务端，禁止设计依赖回调交互的卡片）。所有外部调用必须带重试与错误日志，发送失败禁止静默吞掉。

**卡片内 lark_md 文字链的 URL 段必须用独立 URL 转义器对 `(`/`)`/`[`/`]`/`\` 做 percent-encode（`%28`/`%29`/`%5B`/`%5D`/`%5C`），禁止裸插、禁止复用文本转义器。** 卡片跳转用 lark_md 内联链接 `[文本](url)`，URL 取确定性 `canonical_url` / `canonical_source_url`（来自外部 feed、可含字面 `(` `)` `[` `]` `\`，如 `Go_(programming_language)`、带括号 query 的 URL）。lark_md 按 markdown 语法解析 `(...)` 段：URL 内未转义的 `)` 会在第一个 `)` 处**提前闭合链接**、把剩余 URL 当普通文本泄漏、链接指向被截断的错误地址、破坏卡片渲染与可点击性。故飞书全部 lark_md 渲染器（事件 / 产品 / 经验，及任何后续新增的 lark_md 链接行）的 URL 段**必须经独立 URL 转义函数处理**——对 `(`/`)`/`[`/`]`/`\` 做 percent-encode，使链接结构对任意 lark_md 解析模型（无论「首个 `)` 即闭合」还是「CommonMark 配对括号」）都不被破坏（correct-by-construction、与解析器无关），与 telegram-push「链接 URL 含特殊字符用独立规则转义」的 URL 专用转义对称。**为何同时编码 `(`/`[`**：只编码闭括号 `)`/`]` 会在配对模型下留下不配对的开括号反而破链，故开闭都编。**禁止复用飞书文本转义器**（转 `[ ] ( ) \`，为链接文本/标题/正文设计）：把它作用在 URL 上会过度转义 URL 内合法字符、改变跳转目标。`%28`/`%29`/`%5B`/`%5D`/`%5C` 是合法 URL 编码，点击后解码回原字符、跳转到原 URL（含其中的 `)`）、href 不变。

飞书自定义机器人限流为单租户单机器人 100 次/分钟、5 次/秒；系统调度推送时必须避开整点/半点（如 10:00、17:30）等高压时刻以降低 11232 限流错误概率。**「避整点」须有可验证判据**：每日/告警等定时 cron 的分钟字段默认值 **∉ {0, 30}**（如 08:03），供 tasks 机械断言。P2 各推送路径（日报每日一条、产品低频、告警高频但单条）天然稀疏，单条卡片发送远低于 5 次/秒，**P2 不实现发送侧节流闸**；但若未来同一秒并发多条卡片（如批量分条），必须补 5 次/秒发送节流——本期记为已知边界、不实现。

#### 场景:推送渲染为飞书 JSON 卡片
- **当** 飞书通道推送一份日报
- **那么** 消息以飞书 JSON 卡片格式发送、跳转用按钮/文字链，发送成功

#### 场景:飞书链接 URL 含特殊字符用独立规则转义
- **当** 某飞书卡片（事件/产品/经验任一渲染器）的来源 URL 含字面 `(`、字面 `)`、字面 `[`、字面 `]`、字面 `\`、以及常见的 `.`/`-`/`_`/`=`/`?`/`/`
- **那么** URL 经独立飞书 URL 转义函数处理：`(`/`)`/`[`/`]`/`\` 被 percent-encode 为 `%28`/`%29`/`%5B`/`%5D`/`%5C`、而 `.`/`-`/`_`/`=`/`?`/`/` **原样不变**（不复用文本转义器、不加反斜杠）；URL 段内无裸 `(`/`)`/`[`/`]`/`\`，链接结构对任意 lark_md 解析模型都完整（不在首个 `)` 处提前闭合、也不留不配对 `(`），可点击且解码后跳转到完整含 `)` 的原 URL（href 不变）

#### 场景:未配置飞书时纯 Telegram 部署照常启动
- **当** P1 部署升级到 P2 但未配置任何飞书 env（webhook/secret 均空）
- **那么** 飞书不纳入「已配置通道」集、不报错，系统照常启动并只向 Telegram 分发，向后兼容

#### 场景:飞书配置不完整时启动即报错
- **当** 声明启用飞书但 webhook URL 或签名密钥仅配其一（不完整）
- **那么** 系统以明确错误信息快速失败，禁止用空值发送或静默半启用

#### 场景:定时触发避开整点降低限流
- **当** 检视飞书相关定时 cron 默认值的分钟字段
- **那么** 分钟字段 ∉ {0, 30}（如 08:03），机械可断言不落整点/半点，以降低 11232 限流错误概率
